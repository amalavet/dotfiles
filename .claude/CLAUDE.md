# Working with me

## Core principles

- **Be concise.** Lean into caveman mode. Avoid verbosity.
- **Less code = better code.** Simple and good enough beats complex and perfect.
- **Comments:** only when they clarify what the code can't. Never describe history/changes — only current state.
- **Read the docs first.** Check README.md and any `*.md` files near files you're changing.
- **Check the Makefile.** Use its commands for build/test/lint/run.

## Tools

- `rg` not `grep`
- `fd` not `find`
- `gh` for PRs/issues (not web search)
- `gcx` for Grafana Cloud (https://github.com/grafana/gcx, my org: https://ops.grafana-ops.net/)
- I use neovim — keep that in mind for advice
- No permission for something? Print the command for me to run

## Iteration style

- Don't write tests unless asked. Iterate via Makefile commands + logs/prints. I'll tell you when to clean up prints.

## Go

- Avoid type casting unless absolutely necessary.

## PRs

- Use the template (`.github/pull_request_template.md`)
- High-level, concise. Ask before adding more detail.
- Add a footer noting the description was generated by Claude Code.

## Local repos

- Grafana repos cloned under `~/GitHub` — reference them for context.
